# Урок 5. Практика. Стилизация компонентов через ИИ

## Пререквизиты для всех участников

- Установлен и работает Claude Code или Open Code
- Установлен Git, есть учетная запись GitHub
- Запросить доступ collaborator в репозиторий https://github.com/Amico1285/learning-course-practice (репозиторий публичный, но для прямого пуша веток нужен доступ на запись)

## Задание: стилизация компонентов

### Шаги

1. Клонировать репозиторий, создать отдельную ветку, установить зависимости
2. Скачать переменные из Figma и положить их в папку проекта
3. С помощью AI синхронизировать переменные Bootstrap с переменными из Figma
4. Стилизовать кнопки и инпуты в Storybook
5. Сделать в проекте форму авторизации, используя стилизованные компоненты
6. ** Если успеете: сделать JSON-файл вместо бэкенда и обработать все возможные сценарии и ошибки в форме авторизации

## Задание: скилл для анализа бэклога ** 

### Шаги

1. Организовать доступ к бэклогу -- через CLI или создать простой MCP-сервер
2. Побрейнштормить с Claude: как кластеризировать фичи, оценивать частотность, приоритизировать и делать триаж
3. С помощью Skill Creator создать скилл на основе выработанного подхода
4. Создать наборы Evaluations -- на ограниченном сете (~50 фичей)
5. Проверить глазами наборы Evaluations: отражают ли они ожидаемый результат
6. Протестировать скилл через Skill Creator и сравнить: работает ли со скиллом лучше, чем без

## Задание: перевод технической документации 

Перевод статьи документации Indeed Certificate Manager с русского на английский с проверкой согласованности терминологии.

### Материалы

В папке `translation-task/` приложены:

- `source-article.md` -- исходная статья для перевода ("Мастер настройки", 754 строки)
- `glossary-product.md` -- глоссарий терминов продукта Indeed Certificate Manager (260+ терминов)
- `glossary-security.md` -- глоссарий IT/Security-терминов PKI (80+ терминов)
- `rubric.md` -- строгая рубрика оценки качества перевода (7 критериев, шкала 0-5)

### Шаги

1. Перевести статью с помощью Claude Code, используя оба глоссария как контекст
2. Проверить перевод на согласованность терминов с глоссариями
3. Сделать несколько вариантов перевода разными моделями:
   - Haiku 4.5 (стандартная)
   - Opus 4.6
   - OpenAI 5.4
   - Gemini 3.1 pro
   - несколько китайских
4. Оценить все переводы с помощью Opus 4.6 по рубрике из `rubric.md`
5. Сравнить результаты: какая модель справилась лучше и почему
