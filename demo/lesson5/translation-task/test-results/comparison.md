# Сравнение качества переводов: RU -> EN

Исходный документ: "Мастер настройки" (Indeed Certificate Manager), ~754 строки.

Оценка выполнена отдельными экземплярами Opus 4.6 по единой рубрике (4 критерия, шкала 0-5, взвешенная сумма).

## Анализ

### Облачные модели (Haiku 4.5, Opus 4.6)

Обе модели показали отличное качество (4.5+). Ключевые наблюдения:

- **Глоссарии**: обе модели полностью соблюдают оба глоссария (K1 = 5/5)
- **Смысловая точность**: обе получили 4/5 -- единичные незначительные отклонения
- **Естественность**: Opus 4/5, Haiku 3/5 -- у Haiku больше калек с русского ("container with users", "object with users", нарушения порядка слов)
- Обе модели полностью сохранили форматирование Docusaurus (Tabs, TabItem, :::info, code blocks, якоря)
- **Вариативность оценок**: первый оценщик Haiku дал 4.80, второй -- 4.30. Разброс ~0.5 балла показывает погрешность LLM-оценки

### Локальная модель Qwen 3.5 4B (Ollama)

Итого 2.45 -- граница между "низким" и "удовлетворительным" качеством. Основные проблемы:

- **Грубое несоблюдение глоссариев** (K1 = 2/5): 20+ терминов переведены не по глоссарию. "Setup Wizard" -> "Configuration Wizard", "Management Console" -> "Control Console", "Self-Service Portal" -> "Self-Service Service", "Certificate thumbprint" -> "fingerprint", "CIPF" -> "SCZI" и др.
- **Смысловые искажения** (K2 = 3/5): подмена "encryption" на "decryption" в целом разделе, "отчество" -> "matronym" (вместо "patronymic"), непереведенный русский предлог "с"
- **Подстрочный стиль** (K3 = 2/5): систематическое отсутствие артиклей, кальки ("time existence", "way of agent identification in domain and out of domain"), нечитаемые конструкции
- **Якоря изменены** (K4 = 3/5): все anchor ID заменены на английские, что ломает внутренние ссылки

### Провалившиеся модели

- **Qwen 3.5 0.8B**: галлюцинация названия продукта ("TrueCM" вместо "Indeed CM"), 5527 строк вместо ~750 (модель зациклилась), преамбула "thinking" на 31 строку. Оценка не проводилась -- перевод явно непригоден.
- **Qwen 3.5 2B**: не завершил перевод (только 1.4KB, преамбула "thinking" без реального контента).

## Выводы

1. Облачные модели (Haiku 4.5 и Opus 4.6) справляются с переводом технической документации на отличном/хорошем уровне при условии предоставления глоссариев.
2. Маленькие локальные модели (0.8B, 2B, 4B) не способны качественно выполнить перевод с соблюдением глоссариев. Размер модели критичен: 0.8B галлюцинирует, 2B не справляется с объемом, 4B переводит но игнорирует глоссарии.
3. Глоссарии -- ключевой фактор: при одинаковом промпте облачные модели следуют глоссариям (K1 = 5), а локальные -- нет (K1 = 2).
4. LLM-оценка имеет погрешность ~0.5 балла: два независимых прогона Opus для одного перевода дали 4.80 и 4.30. Для повышения надежности можно запускать несколько оценщиков и усреднять.
5. Для учебного задания рекомендуется сравнивать модели разного класса, чтобы наглядно продемонстрировать разницу в качестве.
