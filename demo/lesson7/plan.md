# Урок 7 -- Практика: GitLab через OpenCode

## Формат

Один час: ~20 мин теория и настройка + ~30 мин практика + ~10 мин кейс.

## Часть 1. Подключение OpenCode к GitLab через glab (~10 мин)

- Что такое glab: CLI для GitLab, аналог gh для GitHub -- агент использует его для работы с issues, merge requests, репозиториями

### Шаг 1. Установка glab

**Вариант A** -- через winget (встроенный пакетный менеджер Windows 10/11):

Открыть PowerShell и выполнить:

```
winget install GLab.GLab
```

**Вариант B** -- если winget недоступен:

Скачать установщик вручную со страницы https://gitlab.com/gitlab-org/cli/-/releases (файл `glab_*_Windows_x86_64.msi`), запустить и следовать инструкциям.

После установки закрыть и снова открыть терминал, чтобы команда `glab` стала доступна.

### Шаг 2. Создание Personal Access Token в GitLab

1. Открыть https://gitlab.com/-/user_settings/personal_access_tokens
2. Нажать **Generate token** -> **Legacy token**
3. Заполнить форму:
   - **Token name** -- любое понятное имя, например `glab-cli`
   - **Expiration date** -- оставить значение по умолчанию (30 дней) или выбрать нужную дату
   - **Select scopes** -- отметить галочку **api** (полный доступ к API, необходим для glab)
4. Нажать **Generate token**
5. Скопировать токен со страницы -- он показывается только один раз

### Шаг 3. Авторизация glab

В PowerShell выполнить:

```
glab auth login --hostname gitlab.com --token ВСТАВИТЬ_ТОКЕН_СЮДА
```

### Шаг 4. Проверка подключения

```
glab auth status
```

В ответе должно быть `Logged in to gitlab.com as ваш_username` -- значит все работает. Попросить агента показать список проектов для дополнительной проверки.
