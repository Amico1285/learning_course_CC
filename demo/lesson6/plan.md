# Урок 6 — Практика: прототипирование

## Формат

Полтора часа: ~20 мин теория + ~1 час практика.

## Теория (~20 мин)

### 1. Показ готового прототипа (~5 мин)

Открыть ранее сделанный прототип, показать:
- Как он выглядит и работает
- Что данные не захардкожены, а живут в стейте
- Как debug-панель показывает текущее состояние

### 2. Три принципа прототипирования с AI (~10 мин)

**Простота для AI:**
- Один сервис со стейтом, понятная архитектура
- Чем проще код, тем лучше AI с ним справляется
- Angular Service + BehaviorSubject вместо NgRx

**Прозрачность для дизайнера:**
- Debug-панель: видишь все переменные прототипа в реальном времени
- Аналогия с переменными в Figma
- Можно редактировать стейт руками и сразу видеть результат

**Переиспользование компонентов:**
- Storybook как единый источник правды для стилей
- Компоненты из Storybook используются в прототипе
- Изменения в Storybook автоматически попадают в прототип

### 3. Архитектура (~5 мин)

- Monorepo: Storybook (компоненты и тема) + Demo_project (прототип)
- Mock-стейт через Angular Service вместо бэкенда
- Hash-роутинг + GitHub Pages = прототип доступен по ссылке
- GitHub Actions = автоматический деплой при push

Показать структуру:
```
repo/
  Storybook/          -- тема и компоненты
  Demo_project/       -- прототип (Angular + Bootstrap)
    src/app/store/    -- весь стейт здесь
    src/app/pages/    -- страницы
    src/app/components/debug-panel/  -- панель отладки
  .github/workflows/  -- автодеплой
```

## Практика (~1 час)

### Подготовка (~10 мин)

1. Клонируйте репозиторий и откройте его в Claude Code / opencode
2. В терминале: `cd Demo_project && npm install && npm start`
3. Откройте прототип в браузере (http://localhost:4200)
4. Кликните на "Demo Project" в навбаре -- откроется debug-панель. Это ваш главный инструмент: здесь видно все данные прототипа

### Как работать с агентом

- Описывайте, что вы хотите видеть и как это должно работать. Не нужно говорить на языке кода.
- После каждой просьбы проверяйте результат в браузере.
- Если что-то сломалось, скажите агенту: "Что-то сломалось, посмотри ошибку и почини"
- Если непонятно, что происходит: "Покажи мне в debug-панели, где я могу увидеть [то, что вас интересует]"
- Если хотите откатить: "Верни как было до последнего изменения"

### Задание (на выбор, ~50 мин)

Выберите одну задачу. Реализуйте ее, общаясь с агентом.

---

**Задача 1: "Роли и права доступа"**

Цель: прототип должен вести себя по-разному в зависимости от того, кто им пользуется.

Опишите агенту:

> "Я хочу, чтобы в навбаре справа можно было выбирать, за какого пользователя я сейчас залогинен. Сделай выпадающий список с именами всех пользователей из таблицы."

Проверьте: в навбаре появился список. Откройте debug-панель -- посмотрите, появился ли текущий пользователь в данных.

> "Теперь сделай так, чтобы интерфейс менялся в зависимости от роли выбранного пользователя. Если я залогинен как Viewer -- я не должен видеть кнопки редактирования и удаления в таблице пользователей. Если как Editor -- вижу только редактирование. Если как Admin -- вижу все."

Проверьте: переключайте пользователя в навбаре и смотрите, как таблица меняется.

Если хотите пойти дальше:
> "Добавь страницу настроек, где можно менять роли пользователей. Ссылку на нее поставь в навбар."

---

**Задача 2: "Онбординг-визард"**

Цель: создать пошаговую форму создания нового пользователя, где на каждом шаге видно, как данные накапливаются.

Опишите агенту:

> "Сделай новую страницу для создания пользователя по шагам. Добавь ссылку на нее в навбар, назови 'Onboarding'. На странице должно быть три шага: первый -- ввод имени и email, второй -- выбор роли, третий -- предпросмотр всех данных и кнопка Создать. Сверху -- полоска прогресса, которая заполняется на каждом шаге. Внизу -- кнопки Назад и Далее."

Проверьте: пройдите по шагам. Откройте debug-панель и посмотрите, сохраняются ли промежуточные данные на каждом шаге.

Если данные не видны в debug-панели:
> "Я хочу видеть в debug-панели, что пользователь ввел на каждом шаге визарда. Сейчас там ничего нет. Сделай так, чтобы при переходе между шагами промежуточные данные сохранялись в стейте."

> "Когда я нажимаю Создать на последнем шаге, новый пользователь должен появиться в таблице на странице Users. После создания -- перенаправь меня туда."

Проверьте: создайте пользователя через визард, перейдите на Users -- он должен быть в таблице.

---

**Задача 3: "Уведомления"**

Цель: чтобы прототип уведомлял о том, что произошло -- добавлен пользователь, изменен статус, удален кто-то.

Опишите агенту:

> "Я хочу систему уведомлений. В навбаре справа -- иконка колокольчика. Рядом с ней -- красный кружок с количеством непрочитанных уведомлений. При клике на колокольчик -- выезжает панель со списком уведомлений."

Проверьте: колокольчик появился, панель открывается (пока пустая).

> "Теперь сделай так, чтобы уведомления появлялись автоматически. Когда добавляется новый пользователь -- уведомление 'Добавлен пользователь [имя]'. Когда меняется статус -- 'Статус [имя] изменен на [новый статус]'. Когда удаляется -- 'Пользователь [имя] удален'."

Проверьте: добавьте пользователя через форму, поменяйте статус в таблице. Бейдж на колокольчике должен расти, а уведомления -- появляться в панели. Откройте debug-панель -- там должен быть массив уведомлений.

> "Добавь возможность пометить уведомление как прочитанное (оно должно стать серым) и кнопку Очистить все внизу панели."

---

**Задача 4: свободная тема**

Придумайте свою фичу и опишите ее агенту. Главное -- она должна менять данные прототипа, и вы должны видеть эти изменения в debug-панели.

Примеры идей:
- Поиск и фильтрация по таблице пользователей
- Темная тема с переключателем
- Страница профиля: кликаешь на пользователя в таблице -- открывается его подробная карточка

---

### Если что-то пошло не так

Не паникуйте. Скажите агенту:

- "Что-то сломалось, приложение не открывается. Посмотри ошибку и исправь."
- "Мне не нравится, как это выглядит. Сделай [опишите, как хотите]."
- "Я не понимаю, что происходит. Объясни мне простыми словами."
- "Где в debug-панели я могу посмотреть [то, что интересует]?"
- "Верни все как было до последнего изменения."

### Демо-сценарий с Playwright (~в конце, 10 мин)

Когда фича готова, запишите демо-сценарий -- автоматический прогон прототипа, который показывает, что получилось.

Скажите агенту:

> "Запиши демо-сценарий с Playwright, который показывает [то, что вы сделали]. Открой прототип, пройди по шагам, покажи результат. Между шагами делай паузу, чтобы зритель успевал видеть."

Например, для задачи с ролями:
> "Запиши демо: открой прототип, переключись на пользователя с ролью Viewer -- покажи, что кнопки пропали. Потом переключись на Admin -- покажи, что кнопки появились."

Агент создаст файл в `e2e/` и запустит его. Браузер откроется и выполнит сценарий с записью видео.

### Финал

1. Скажите агенту: "Сделай коммит и запуши"
2. GitHub Actions автоматически опубликует прототип
3. Каждый покажет свой результат: ссылка на GitHub Pages + демо-видео

## Подготовка перед уроком

Студенты должны установить у себя:
- MCP-сервер Playwright (для Claude Code / opencode -- чтобы агент мог управлять браузером)
- `npx playwright install chromium` в корне репозитория после клонирования
