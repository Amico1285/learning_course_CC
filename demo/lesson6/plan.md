# Урок 6 — Практика: прототипирование

## Формат

Полтора часа: ~20 мин теория + ~1 час практика.

## Теория (~20 мин)

### 1. Показ готового прототипа (~5 мин)

Открыть ранее сделанный прототип, показать:
- Как он выглядит и работает
- Что данные не захардкожены, а живут в стейте
- Как debug-панель показывает текущее состояние

### 2. Три принципа прототипирования с AI (~10 мин)

**Простота для AI:**
- Один сервис со стейтом, понятная архитектура
- Чем проще код, тем лучше AI с ним справляется
- Angular Service + BehaviorSubject вместо NgRx

**Прозрачность для дизайнера:**
- Debug-панель: видишь все переменные прототипа в реальном времени
- Аналогия с переменными в Figma
- Можно редактировать стейт руками и сразу видеть результат

**Переиспользование компонентов:**
- Storybook как единый источник правды для стилей
- Компоненты из Storybook используются в прототипе
- Изменения в Storybook автоматически попадают в прототип

### 3. Архитектура (~5 мин)

- Monorepo: Storybook (компоненты и тема) + Demo_project (прототип)
- Mock-стейт через Angular Service вместо бэкенда
- Hash-роутинг + GitHub Pages = прототип доступен по ссылке
- GitHub Actions = автоматический деплой при push

Показать структуру:
```
repo/
  Storybook/          -- тема и компоненты
  Demo_project/       -- прототип (Angular + Bootstrap)
    src/app/store/    -- весь стейт здесь
    src/app/pages/    -- страницы
    src/app/components/debug-panel/  -- панель отладки
  .github/workflows/  -- автодеплой
```

## Практика (~1 час)

### Подготовка (~10 мин)

Студенты клонируют репозиторий, переключаются на ветку `lesson6/prototyping`.
Запускают `npm install` и `npm start` в папке Demo_project.
Открывают прототип в браузере, кликают на лого "Demo Project" -- видят debug-панель.

### Задание (на выбор, ~50 мин)

Студенты выбирают одну задачу и реализуют ее с помощью Claude Code.

Общие советы перед началом:
- Откройте debug-панель (клик на "Demo Project") и держите ее открытой -- она покажет, что происходит "под капотом"
- Не пишите код руками. Описывайте Claude Code, что хотите, на человеческом языке
- Если что-то сломалось -- скажите Claude Code "не компилируется, посмотри ошибку" или "верни как было"
- После каждого шага проверяйте результат в браузере

---

**Задача 1: "Роли и права доступа"**

Цель: сделать так, чтобы прототип вел себя по-разному для разных пользователей.

Что должно получиться:
- В навбаре справа -- выпадающий список, где можно выбрать, за какого пользователя ты "залогинен"
- Если выбран Viewer -- на странице Users не видно кнопок Edit и Delete
- Если выбран Editor -- видно Edit, но не Delete
- Если выбран Admin -- видно все
- В debug-панели видно, какой currentUser сейчас выбран

Пошаговые подсказки:

Шаг 1. Скажите Claude Code:
> "Добавь в стейт поле currentUser, которое по умолчанию равно первому пользователю из списка users. Добавь метод setCurrentUser(id) в сервис AppStoreService."

Проверьте: откройте debug-панель, в JSON должно появиться поле currentUser.

Шаг 2. Скажите Claude Code:
> "В навбаре справа добавь Bootstrap-dropdown, который показывает имя текущего пользователя. При клике на элемент списка -- переключает currentUser в стейте. Используй данные из store.users$."

Проверьте: в навбаре появился выпадающий список, при переключении в debug-панели меняется currentUser.

Шаг 3. Скажите Claude Code:
> "На странице Users скрой кнопку Delete для всех, кроме Admin. Скрой кнопку Edit для Viewer. Используй роль из currentUser."

Проверьте: переключите пользователя в навбаре -- кнопки появляются и исчезают.

---

**Задача 2: "Онбординг-визард"**

Цель: создать пошаговую форму, где видно, как стейт меняется на каждом шаге.

Что должно получиться:
- Новая страница /onboarding (ссылка в навбаре)
- Форма из 3 шагов: ввод имени и email -> выбор роли -> подтверждение
- Прогресс-бар вверху показывает, на каком шаге находишься
- После нажатия "Создать" -- пользователь появляется в списке Users
- В debug-панели видно, как на каждом шаге добавляются данные

Пошаговые подсказки:

Шаг 1. Скажите Claude Code:
> "Создай новую страницу OnboardingComponent по пути src/app/pages/onboarding/. Добавь маршрут /onboarding в app.routes.ts и ссылку 'Onboarding' в навбар в app.html."

Проверьте: в навбаре появилась новая ссылка, при клике открывается пустая страница.

Шаг 2. Скажите Claude Code:
> "На странице Onboarding сделай визард из 3 шагов. Шаг 1: поля Name и Email. Шаг 2: выбор Role (Admin/Editor/Viewer) через radio-кнопки. Шаг 3: показать все введенные данные и кнопку Создать. Сверху -- Bootstrap progress bar, который заполняется на каждом шаге (33%, 66%, 100%). Кнопки Назад и Далее для навигации между шагами."

Проверьте: визард работает, можно переходить между шагами.

Шаг 3. Скажите Claude Code:
> "Добавь в стейт поле onboardingDraft, куда сохраняются промежуточные данные визарда на каждом шаге. Когда пользователь нажимает Далее -- обновляй onboardingDraft в стейте. Когда нажимает Создать -- вызови addUser() и очисти onboardingDraft."

Проверьте: откройте debug-панель и пройдите визард. В JSON видно, как onboardingDraft заполняется пошагово.

---

**Задача 3: "Уведомления"**

Цель: добавить систему уведомлений, которая реагирует на действия в прототипе.

Что должно получиться:
- В навбаре -- иконка колокольчика с красным бейджем (количество непрочитанных)
- При клике на колокольчик -- выезжает панель с уведомлениями
- Когда кто-то добавляет пользователя, меняет статус или удаляет -- автоматически появляется уведомление
- Можно пометить уведомление как прочитанное или очистить все
- В debug-панели видно массив notifications

Пошаговые подсказки:

Шаг 1. Скажите Claude Code:
> "Добавь в стейт массив notifications. Каждое уведомление: id, message (текст), read (прочитано или нет), createdAt (время). Добавь в сервис методы: addNotification(message), markAsRead(id), markAllAsRead(), clearNotifications()."

Проверьте: в debug-панели появился пустой массив notifications.

Шаг 2. Скажите Claude Code:
> "Измени методы addUser, deleteUser, toggleUserStatus в AppStoreService: после каждого действия автоматически вызывай addNotification с описанием того, что произошло. Например, 'User Alice added' или 'User Bob status changed to Inactive'."

Проверьте: добавьте пользователя через форму, откройте debug-панель -- в notifications появилась запись.

Шаг 3. Скажите Claude Code:
> "В навбаре справа добавь кнопку-иконку колокольчика. Рядом -- Bootstrap badge с количеством непрочитанных уведомлений. При клике открывается Bootstrap offcanvas справа со списком уведомлений. У каждого уведомления -- кнопка 'прочитать'. Внизу -- кнопка 'Очистить все'."

Проверьте: выполните несколько действий (добавьте пользователя, поменяйте статус) -- бейдж растет, уведомления появляются в панели.

---

**Задача 4: свободная тема**

Если у вас есть своя идея -- реализуйте ее. Главное условие: фича должна использовать стейт (добавлять/менять данные), и изменения должны быть видны в debug-панели.

Примеры идей:
- Фильтрация и поиск по таблице Users
- Темная тема с переключателем (сохранение выбора в стейте)
- Страница профиля пользователя с детальной информацией

- "Верни последнее изменение, оно сломало приложение"

### Демонстрация результатов (~в конце)

Студенты делают коммит и пуш. GitHub Actions автоматически деплоит на Pages.
Каждый показывает свой прототип по ссылке.
