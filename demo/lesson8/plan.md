# Урок 8 -- Практика: правила линтера + управление задачами через Kaiten MCP

## Формат

Полтора часа: ~40 мин правила линтера + ~10 мин настройка Kaiten + ~30 мин практика + ~10 мин обсуждение.

---

## Часть 1. Правила линтера от техписов (~40 мин)

### Контекст (~5 мин)

- Техписы курса подготовили набор правил для линтера, которые проверяют тексты на соответствие стайлгайду
- Правила автоматически находят типичные ошибки: канцеляризмы, чрезмерная вежливость, неправильные кавычки, лишние пробелы и т.д.
- Это пример того, как команда техписов может передать свои знания в форме, которую ИИ-агент может использовать автоматически

### Разбор правил (~15 мин)

Показать студентам:
- Где лежат правила (файлы конфигурации линтера)
- Как устроено одно правило: паттерн, сообщение об ошибке, пример
- Какие категории ошибок покрыты (стиль, пунктуация, терминология, форматирование)
- Как правила связаны со стайлгайдом: каждое правило -- это формализованная рекомендация из гайда

### Практика: запуск линтера (~20 мин)

1. Открыть OpenCode в папке с текстовыми файлами
2. Попросить агента проверить текст по правилам:

> "Проверь этот текст на соответствие нашему стайлгайду. Используй правила линтера."

3. Посмотреть, какие ошибки найдены
4. Попросить агента исправить:

> "Исправь найденные ошибки, но сохрани смысл текста."

5. Сравнить до и после -- обсудить, согласны ли с исправлениями

## Часть 2. Настройка Kaiten MCP (~10 мин)

### Что такое Kaiten MCP (~2 мин)

Kaiten -- система управления задачами (как Jira или Trello). MCP-сервер позволяет агенту работать с задачами в Kaiten напрямую: создавать карточки, двигать по колонкам, оставлять комментарии, логировать время -- все через текстовые команды.

63 инструмента: карточки, комментарии, чеклисты, теги, учет времени, блокеры, внешние ссылки, файлы и др.

### Шаг 1. Создание API-токена в Kaiten (~3 мин)

1. Открыть ваш Kaiten (например, `https://your-company.kaiten.ru`)
2. Нажать на аватар в правом верхнем углу -> **Настройки профиля**
3. В левом меню выбрать **API Key**
4. Нажать **Создать токен**
5. Скопировать токен -- он показывается только один раз

### Шаг 2. Узнать ID пространства и доски (~2 мин)

ID видны прямо в адресной строке браузера:

- **Space ID** -- откройте пространство, посмотрите URL: `https://your-company.kaiten.ru/space/762572` -- space ID = `762572`
- **Board ID** -- откройте доску: `https://your-company.kaiten.ru/space/762572/boards/1727446` -- board ID = `1727446`

Эти ID понадобятся при работе с агентом (он будет спрашивать, на какой доске создавать карточки).

### Шаг 3. Добавить MCP-сервер в OpenCode (~3 мин)

Открыть файл `opencode.json` в корне проекта (или создать его, если нет) и добавить:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "kaiten": {
      "type": "local",
      "command": ["npx", "-y", "kaiten-mcp"],
      "enabled": true,
      "environment": {
        "KAITEN_API_TOKEN": "ваш-токен-из-шага-1",
        "KAITEN_URL": "https://your-company.kaiten.ru"
      }
    }
  }
}
```

Для **Claude Code** конфиг выглядит иначе -- файл `.mcp.json`:

```json
{
  "mcpServers": {
    "kaiten": {
      "command": "npx",
      "args": ["-y", "kaiten-mcp"],
      "env": {
        "KAITEN_API_TOKEN": "ваш-токен-из-шага-1",
        "KAITEN_URL": "https://your-company.kaiten.ru"
      }
    }
  }
}
```

Перезапустить OpenCode (или Claude Code) после изменения конфига.

### Шаг 4. Добавить инструкции для агента

Чтобы агент лучше понимал, как работать с Kaiten, добавьте в файл `AGENTS.md` в корне проекта (для OpenCode) или `CLAUDE.md` (для Claude Code):

```
Read ./node_modules/kaiten-mcp/LLM_GUIDE.md before working with Kaiten.
```

Эта строка говорит агенту прочитать подробный гайд перед первым использованием инструментов Kaiten. В гайде описана иерархия объектов, типовые сценарии, особенности API и частые ошибки.

### Шаг 5. Проверка подключения

Попросить агента:

> "Покажи список пространств в Kaiten."

Если все настроено правильно, агент вызовет `kaiten_list_spaces` и покажет ваши пространства.

## Часть 3. Практика: управление задачами через агента (~30 мин)

### Задание

Работаем на реальной доске в Kaiten. Каждый студент выполняет сценарий через агента.

### Сценарий 1: Создание и ведение задачи (~15 мин)

> "Покажи мне доски в пространстве [ID]. Потом покажи колонки на доске [ID]."

Студент видит структуру своего рабочего пространства.

> "Создай карточку на доске [ID] в колонке Очередь. Заголовок: 'Обновить раздел FAQ на сайте'. Описание: нужно проверить актуальность всех вопросов и добавить раздел про API. Размер: 3 SP. Дедлайн: через неделю."

> "Добавь к этой карточке чеклист 'Шаги' с пунктами: 1) Проверить текущие вопросы, 2) Удалить неактуальные, 3) Написать раздел про API, 4) Согласовать с командой."

> "Добавь тег 'documentation' и оставь комментарий: 'Взял в работу, начну с аудита текущих вопросов'."

> "Перемести карточку в колонку В работе."

> "Отметь первый пункт чеклиста как выполненный и залогируй 30 минут работы."

### Сценарий 2: Обзор доски и отчетность (~15 мин)

> "Покажи все карточки на доске [ID] с нормальной детализацией."

> "Найди все карточки с тегом 'documentation'."

> "Покажи, сколько времени залогировано на доске [ID] за сегодня."

> "Покажи историю перемещений карточки [ID] -- в каких колонках она была и когда."

Для продвинутых:

> "Создай блокер на карточке [A]: она заблокирована карточкой [B] с причиной 'Ждем утверждения дизайна'. Когда карточка [B] будет в Готово, блокер снимется автоматически."

## Часть 4. Обсуждение (~10 мин)

- Какие задачи удобно делать через агента, а какие быстрее в UI?
- Сценарии для команд: один человек создает задачи голосом через агента, другой видит их на доске в реальном времени
- Автоматизация рутины: еженедельные отчеты по времени, создание типовых карточек, аудит просроченных задач

---

## Подготовка перед уроком

Студенты должны:

1. Иметь доступ к Kaiten (инстанс компании или учебный)
2. Создать API-токен в Kaiten (Профиль -> API Key)
3. Записать URL инстанса, space ID и board ID из адресной строки
4. Добавить Kaiten MCP-сервер в конфиг OpenCode / Claude Code (см. Шаг 3)
5. Проверить подключение командой "Покажи список пространств в Kaiten"
6. Иметь файлы текстов для проверки линтером (будут предоставлены)

### Различия конфигурации OpenCode и Claude Code

| Параметр | OpenCode | Claude Code |
|----------|----------|-------------|
| Файл конфига | `opencode.json` | `.mcp.json` |
| Ключ для серверов | `"mcp"` | `"mcpServers"` |
| Формат команды | `"command": ["npx", "-y", "kaiten-mcp"]` | `"command": "npx"` + `"args": ["-y", "kaiten-mcp"]` |
| Переменные окружения | `"environment": {...}` | `"env": {...}` |
| Файл инструкций | `AGENTS.md` | `CLAUDE.md` |
| Глобальный конфиг | `~/.config/opencode/opencode.json` | `~/.claude/.mcp.json` |
